# Build a Visual PDF Template Configuration System for Excel-to-PDF Banking Reports

## Context

We already have a working application that:

* uploads Excel files
* processes multiple sheets
* generates PDFs

However, the current system is heavily hardcoded.

Problems in current system:

* PDF layout is coded manually
* adding/removing columns requires code changes
* changing row/column sizes requires code changes
* grouping logic is hardcoded
* summary sections are hardcoded
* bank-specific layouts require engineering effort
* mappings between Excel sheets and PDF are static
* many business rules are embedded directly in backend code

We now want to build a fully dynamic, visual, configuration-driven system.

The goal is:

> Non-technical users should be able to visually configure PDF layouts, mappings, grouping logic, table structure, summary sections, and rendering rules without modifying code.

This is NOT a generic spreadsheet app.

This is a:

# Visual Banking PDF Template Designer

---

# Core System Requirements

The system should allow users to:

* upload Excel files containing multiple sheets
* preview all sheets visually
* select columns from different sheets
* bind selected columns into PDF sections
* visually design PDF layouts
* resize rows/columns visually
* configure table structure visually
* define grouping logic
* define summary headers
* generate PDFs dynamically
* save reusable templates
* support manual blank columns in PDF
* support repeated/grouped sections
* support branch-wise PDF generation
* support complex business rules

---

# High-Level Architecture

The system should have 5 major modules:

1. Excel Data Engine
2. Data Mapping Layer
3. Visual Template Designer
4. Rule & Grouping Engine
5. Dynamic PDF Rendering Engine

---

# 1. Excel Data Engine

## Requirements

The system must:

* upload `.xlsx` and `.csv`
* support multiple sheets
* preserve original sheet names
* preview sheet data
* normalize rows internally
* track source metadata

Each cell should internally contain:

```ts
{
  sheetName,
  rowIndex,
  columnIndex,
  columnName,
  value
}
```

## UI Requirements

### Left Sidebar

Show:

* uploaded file
* all sheets
* all columns

Workbook
├── Sheet1
│    ├── Name
│    ├── Amount
│    ├── Branch
│
├── Sheet2
│    ├── GoldType
│    ├── Weight

# 2. Data Mapping Layer

This is the most important layer.

We should NOT directly render Excel rows into PDF.

Instead:

Excel → Internal Data Model → PDF Renderer

Example normalized model:

```ts
{
  branch,
  customerName,
  accountNo,
  ornaments[],
  totalAmount
}
```

The UI must allow users to visually bind data.

Example:

* drag column from sheet
* drop into PDF field

OR

click field → select Excel column

Example:

"Customer Name" → Sheet1.Name
"Branch" → Sheet1.Branch
"Gold Type" → Sheet2.GoldType

# Manual Blank Columns

Important feature:

Users sometimes intentionally leave columns blank in generated PDF for handwritten entries.

Example:

| Customer | Amount | Signature | Remarks |

Where:

* Signature
* Remarks

are blank intentionally.

The designer must support static/manual columns.

---

# Dynamic Data Fields

Support placeholders like:

```txt
{{branch_name}}
{{customer_name}}
{{total_amount}}
```

These should bind dynamically.

---

# Summary Section Requirements

Support configurable summary headers.

Example:

Branch: {{branch}}
Total Accounts: {{count}}
Total Amount: {{sum}}

These should support:

* fixed positioning
* page repetition
* conditional rendering

---

# 4. Rule & Grouping Engine

The system must support visual business rules.

Do NOT hardcode rules.

Rules should be configurable.

---

# Required Rule Types

## A. Grouping Rules

Examples:

Group by Branch
Generate separate PDF per branch

OR

Group by Account Number

---

## B. Merge/Duplicate Rules

Example:

Same customer appears multiple times due to:

* multiple ornaments
* multiple entries

Assign same serial number.

---

## C. Filtering Rules

Examples:

Ignore rows where:

* amount = 0
* branch is empty
* customer name missing

---

## D. Aggregation Rules

Examples:

* total amount
* count customers
* total ornaments

---

## E. Conditional Rendering

Examples:

If ornaments exist → show section
Else hide section

---

# Rule Builder UI

The UI should be visual.

Example:

IF
[Amount]
[ > ]
[1000]

THEN
[Highlight Row]

---

Support:

Users must be able to:

* save template
* edit template
* duplicate template
* preview template
* export template
